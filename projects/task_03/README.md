# Корпоративная сеть

Необходимо реализовать корпоративную сеть с доступом из внешнего интернета и с доступом во внешний интернет:

## Требования к реализации

- На входе в сеть стоит anycast - пользователь обращается к физически ближайшему узлу входа в сеть (Edge node)
- На Edge node стоит L3/L4 балансировщик, равномерно распределяя запросы на L7 уровень
- За Edge node находится L7 балансироващик, который распределяет HTTP запросы
- Далее стоит Firewall с политиками безопасности (доступы к БД, SSH из внешней сети)
- Необходимо настроить NAT из внутренней сети для доступа в интернет (например, для установки пакетов из внешних registry) 
- Реализовать доступ по SSO + SSH во внутреннюю сеть (используйте готовые инструменты)
- Реализовать DNS в сети: itmo.ru - для запросов из внешней сети без SSO на публичные ресурсы, itmo-team.ru - для запросов из внутренней сети на внутренние ресурсы
- Все сервисы или часть сервисов (это должно быть сложнее) должны работать на IPv6

## Пользовательский интерфейс

Внутри сети должно быть развернуто несколько сервисов:

- Несколько инстансов одного веб-сервера (доступ из публичной сети, https://itmo.ru)
- База данных (доступ из внутренней сети только через интерфейс, доступ только из внутренней сети)
- Несколько веб-серверов с интерфесом к базе данных (DbGate, CloudBeaver, самописное что-то; доступ через SSO - https://itmo-team.ru)

## Комментарии
 
- Можно пользоваться любыми библиотеками, утилитами и инструментами
- Должно быть проще разворачивать все через containerlab, но это не точно
- Anycast и все-все можно эмулировать просто доступом из других контейнеров Docker. Использовать можно маршрутизацию OSPF / BGP / IS-IS. Предполагаем, что доступ из публичной сети - это соседние контейнеры, а не реальный интернет. В пункте про NAT - желательно пробросить доступ в глобальный интернет
- Если есть проблемы с реализацией какого-либо пункта, необходимо реализовать заглушку / написать собственную реализацию / пропустить пункт - но описать, почему именно такое решение было принято и его недостатки на защите
