# Распределенные вычисления

## Требования к проекту

В проекте нужно реализовать систему, состоящую из одного общего gateway для пользователей и нескольких воркеров. Система должна уметь принимать запрос от пользователя, находить свободный сервер, поднимать там изолированный контейнер с вычислителем и пробрасывать сетевой маршрут от gateway до этого конкретного контейнера.

Публичный IP существует только на gateway, он принимает запросы от пользователя, хранит состояние системы, занимается маршрутизацией трафика, открывает новые порты.

Во внутренней сети находятся сервера, которые выполняют пользовательские задачи. Необходимо реализовать Service Discovery.

## Пользовательский интерфейс

Глобально интерфейс пользователя такой:

- пользователь приходит в gateway и создает задачу (само содержание задачи указано в коде / конфиге сервера, пользователь управляет не содержимым задачи, а только ее процессом выполнения)
- сервер открывает порт на gateway, создает процесс на одном из worker
- пользователь подключается по TCP/UDP к worker на выделенный порт (на IP gateway, а он проксирует) и выполняет некоторые захардкоженные на сервере команды для управления задачей

``` POST /api/jobs ``` - создать новый процесс, в ответе возвращается выделенный порт и протокол по которому выполнить подключение (TCP/UDP)
``` DELETE /api/jobs/{job_id} ``` - убить существующий процесс
``` GET /api/jobs/{job_id} ``` - получить статус процесса

Интерфейс взаимодействия worker и пользователя по TCP/UDP взять произвольный (для простоты, можно просто взять ping-pong с нагрузкой на CPU).

## Комментарии к реализации

- Можно использовать как готовые инструменты, так и писать собственную реализацию для всех. 
- Обязательно нужно подключить инструменты мониторинга и провести нагрузочное тестирование (для демонстрации корректности распределения по воркерам и service discovery).
- Нужен письменный отчёт и инструкция к запуску (можно не очень подробно).
- Обязательно прикрепить подробное видео-демонстрацию работы сервиса со всеми существенными комментариями, можно скинуть в личные сообщения
