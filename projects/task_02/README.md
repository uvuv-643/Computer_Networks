# Федеративное обучение

Федеративное обучение - метод машинного обучения, при котором модель обучается на множестве устройств или серверов, содержащих локальные данные, без обмена этими данными.

Чтобы не смещать фокус проекта от компьютерных сетей для решения задачи реализуем обучение простой нейронной сети для классификации изображений (например, можно взять два MLP слоя с MNIST изображениями). Реализация - любая (самописный, Keras, Torch, ...)

## Пользовательский интерфейс

### Сервер

Сервер работает постоянно, принимает запросы от клиентов, сервер имеет может обучать несколько алгоритмов параллельно. Клиенты на уровне протокола выбирают, какую задачу они считают.

### Клиент

Клиент запускается по запросу, выполняет вычисления и процесс на клиенте умирает.

## Требования к реализации

Необходимо реализовать свой протокол поверх TCP/UDP, обосновать выбор, написать спецификацию протокола.

- Нужен письменный отчёт и инструкция к запуску (можно не очень подробно).
- Обязательно прикрепить подробное видео-демонстрацию работы сервиса со всеми существенными комментариями, можно скинуть в личные сообщения
- Deadline пулл-реквеста - 25.05.2026

## Усложнение (любые 2 из 3)

1) Реализовать прозрачный фреймворк для пользователя (автоматическое определение узлов в сети, выбор сервера среди доверенных устройств)
2) Фреймворк, не зависящий от архитектуры ML-модели (PyTorch только), нужно включить также передачу архитектуры модели по сети
3) На уровне инфраструктуры предложить решение, чтобы сервер не был единственной точкой отказа системы
